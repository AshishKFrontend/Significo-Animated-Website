# Significo - Human-Centric Health Software Landing Page

A faithful clone of a modern, human-centric health software company's landing page, focusing on high-quality design, interactive scrolling experiences, and performance-driven animations.

The project is built using a modern front-end stack, leveraging utility-first CSS and powerful JavaScript libraries for advanced scroll-based visual effects.

## ‚ú® Features

* **Responsive Design:** Fully responsive layout using **Tailwind CSS**.
* **Scroll-Based Video Animation:** Interactive video clip-path animation on the hero section using **GSAP**'s ScrollTrigger.
* **Marquee Text Effect:** Animated, image-inclusive marquee elements that scroll in opposite directions.
* **Horizontal Scroll Section:** An immersive "Real talk, Real impact" section with horizontal scrolling using **GSAP** and **Locomotive Scroll**.
* **Dynamic Theme Change:** The background and text color of the entire page dynamically change as the user scrolls into different sections (`data-color` attribute and custom CSS variables).
* **Interactive Team List:** Hover effects on the team list to reveal member pictures using mouse position tracking.
* **Capsule Parallax Effect:** Custom parallax scrolling for the "Explore Our Insights" capsules.
* **Smooth Scrolling:** Implemented using **Locomotive Scroll** for a highly performant and customized scroll experience.

## üíª Installation and Setup

To run this project locally, follow these steps:

1.  **Clone the repository:**
    ```bash
    git clone [https://github.com/YourUsername/Significo-Health-Software-Landing-Page.git](https://github.com/YourUsername/Significo-Health-Software-Landing-Page.git)
    ```
2.  **Navigate to the project directory:**
    ```bash
    cd Significo-Health-Software-Landing-Page
    ```
3.  **Open `index.html`:**
    Simply open the `index.html` file in your preferred web browser. Since it uses CDN links for Tailwind and GSAP, no build step is required!

    *(Note: For the video and image assets to display correctly, ensure you have placed your own `Video.mp4` and images (`a1.jpg`, `c1.jpeg`, etc.) in the project root or update the paths in `index.html`.)*
